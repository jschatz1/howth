Quick Start

# Install (coming soon)
cargo install howth

# Build your project
howth build

# Watch mode (transpile-only for speed)
howth build --watch

# Bundle for production
howth bundle src/index.ts -o dist/bundle.js

# Dev server with HMR
howth dev src/index.ts --port 3000

Build System

Parallel Hashing

Content-based caching with parallel file hashing. Only rebuild what changed.

Lazy Fingerprinting

Skip expensive output fingerprinting on first build. Computed lazily on cache hits.

Watch Mode

Transpile-only by default for instant feedback. Typecheck is opt-in.

SWC Backend

Uses SWC for transpilation. Rust-native, no Node.js overhead.

Bundler

Tree Shaking

Dead code elimination. Only include what's actually used in your bundle.

Code Splitting

Automatic chunk splitting on dynamic imports. Lazy load routes and features.

Dev Server + HMR

Built-in dev server with hot module replacement. See changes instantly.

CSS + Assets

Bundle CSS with minification. Copy assets with content hashes for caching.

Plugin System

Rollup-compatible plugins. Built-in: replace, alias, banner, virtual modules.

Multiple Formats

Output ESM, CommonJS, or IIFE. Target any JavaScript environment.

Philosophy

howth and Bun share the same philosophy: in watch mode, transpile-only is the default for instant feedback. Type checking is opt-in (--watch typecheck) because correctness can wait for a separate terminal or CI.
